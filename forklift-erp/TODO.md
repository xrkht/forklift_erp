# 开发路线图 (Roadmap)

本文档记录叉车进销存ERP系统的开发规划、功能设想和改进方向。

##  短期目标（1-2周）

### 高优先级
- [ ] 实现全局异常处理机制
- [ ] 添加数据验证注解（@Valid）
- [ ] 创建统一API响应格式类

##  中期目标（1-2个月）

### 业务功能扩展
- [ ] **用户认证与授权**
    - [ ] 集成 Spring Security
    - [ ] JWT Token 认证
    - [ ] 角色权限管理（管理员、销售员、库管员）

- [ ] **采购管理模块**
    - [ ] 采购订单管理
    - [ ] 供应商管理
    - [ ] 采购入库流程

- [ ] **销售管理模块**
    - [ ] 销售订单管理
    - [ ] 客户管理
    - [ ] 销售出库流程
    - [ ] 发票管理

- [ ] **库存管理增强**
    - [ ] 库存盘点功能
    - [ ] 库存预警（低库存提醒）
    - [ ] 库存调拨
    - [ ] 批次管理
    - [ ] 图形化库存管理(此模块中优先级较高)

- [ ] **租赁管理模块**(需完善)
    - [ ] 销售统计报表

- [ ] **报表统计**
    - [ ] 销售统计报表
    - [ ] 库存周转率分析
    - [ ] 利润分析
    - [ ] 导出 Excel 功能

### 技术优化(暂定)

- [ ] 配置数据库连接池（HikariCP 优化）
- [ ] 添加操作日志记录
- [ ] 实现数据备份与恢复
- [ ] Docker 容器化部署

##  长期愿景（3-6个月）

### 系统增强

- [ ] **移动端适配**
    - 开发微信小程序或移动APP
    - 扫码录入叉车信息
